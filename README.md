A collection of my solutions to challenges from **[SadServers](https://sadservers.com/)**. It is a hands-on platform for practicing linux troubleshooting, DevOps, and Site Reliability Engineering (SRE) skills through real-world incident scenarios. I stumbled upon this through a [LinkedIn post](https://www.linkedin.com/posts/rohankumarrshah_sadservers-linux-devops-sre-labs-job-activity-7437512760135090177-Redc?utm_source=share&utm_medium=member_desktop&rcm=ACoAACz4uskB2XO5G9vSUh8fIUov-3SnYHsIGT8).

## Tools & Technologies

Common tools used across the solutions include:

* **Linux CLI utilities**
* Bash scripting
* Process inspection tools (`ps`, `top`, `htop`)
* File and disk utilities (`df`, `du`, `lsof`, `fuser`)
* Networking tools (`ss`, `netstat`, `curl`, `ping`)
* System monitoring and debugging commands
